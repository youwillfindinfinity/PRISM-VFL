# Data Setup

PRISM-VFL experiments require MIMIC-III Clinical Database v1.4. The raw data and derived
files are not included in this repository due to the PhysioNet Data Use Agreement. This
document covers access, download, and preprocessing from scratch.

All commands below assume you are working from the `PRISM-VFL/` root directory.

---

## Expected directory layout after setup

```
data/
├── mimic3-benchmarks/         <- YerevaNN pipeline output
│   └── data/
│       ├── root/              <- per-subject episode CSVs
│       ├── in-hospital-mortality/
│       │   ├── train/
│       │   ├── train_listfile.csv
│       │   ├── val_listfile.csv
│       │   └── test_listfile.csv
│       ├── decompensation/    (same layout)
│       ├── length-of-stay/    (same layout, not used in training)
│       └── phenotyping/       (same layout)
└── vertical_splits/           <- generated by this repo's data_prep scripts
    ├── site_A_vitals.csv
    ├── site_B_labs.csv
    ├── site_C_composite.csv
    └── aligned_patient_ids.csv
```

---

## Step 1 — PhysioNet access

1. Register at https://physionet.org/register/ (institutional email preferred).
2. Complete the CITI "Data or Specimens Only Research" training at
   https://about.citiprogram.org/ (approximately 2–3 hours).
3. Upload your CITI completion certificate to your PhysioNet profile under **Training**.
4. Go to https://physionet.org/content/mimiciii/1.4/ and click **Request access** to sign
   the Data Use Agreement electronically. Approval is typically instant after CITI is
   verified.

---

## Step 2 — Download MIMIC-III

```bash
mkdir -p data/mimic-raw
cd data/mimic-raw

# Replace USERNAME with your PhysioNet username.
# CHARTEVENTS.csv is ~33 GB; use tmux or nohup for large files on a remote machine.
wget -r -N -c -np \
    --user USERNAME --ask-password \
    https://physionet.org/files/mimiciii/1.4/
```

Disk: approximately 40 GB compressed, 35 GB after extraction.

---

## Step 3 — Run the YerevaNN benchmark pipeline

```bash
git clone https://github.com/YerevaNN/mimic3-benchmarks.git
cd mimic3-benchmarks
pip install -r requirements.txt

MIMIC_RAW=../mimic-raw/physionet.org/files/mimiciii/1.4

# Extract per-subject timeseries (30–60 min)
python -m mimic3benchmark.scripts.extract_subjects \
    $MIMIC_RAW ../data/mimic3-benchmarks/data/root/

# Remove implausible values
python -m mimic3benchmark.scripts.validate_events \
    ../data/mimic3-benchmarks/data/root/

# Extract per-ICU-stay episodes
python -m mimic3benchmark.scripts.extract_episodes_from_subjects \
    ../data/mimic3-benchmarks/data/root/

# Create train / test split
python -m mimic3benchmark.scripts.split_train_and_test \
    ../data/mimic3-benchmarks/data/root/

# Create task datasets
python -m mimic3benchmark.scripts.create_in_hospital_mortality \
    ../data/mimic3-benchmarks/data/root/ \
    ../data/mimic3-benchmarks/data/in-hospital-mortality/

python -m mimic3benchmark.scripts.create_decompensation \
    ../data/mimic3-benchmarks/data/root/ \
    ../data/mimic3-benchmarks/data/decompensation/

python -m mimic3benchmark.scripts.create_length_of_stay \
    ../data/mimic3-benchmarks/data/root/ \
    ../data/mimic3-benchmarks/data/length-of-stay/

python -m mimic3benchmark.scripts.create_phenotyping \
    ../data/mimic3-benchmarks/data/root/ \
    ../data/mimic3-benchmarks/data/phenotyping/

# Create val splits for each task
cd ..
for task in in-hospital-mortality decompensation length-of-stay phenotyping; do
    python mimic3-benchmarks/mimic3models/split_train_val.py \
        data/mimic3-benchmarks/data/$task/
done
```

Expected output: approximately 33,798 unique patients, 42,276 ICU stays across four
task directories.

---

## Step 4 — Create the vertical split

Run from the `PRISM-VFL/` root:

```bash
python data_prep/vertical_split.py \
    --root data/mimic3-benchmarks/data/ \
    --output data/vertical_splits/
```

This partitions 14 of the 17 YerevaNN "ready" variables across three simulated hospital
sites. Three GCS sub-scores (eye, motor, verbal) are excluded because GCS total = eye +
motor + verbal; including them at a different site would allow cross-site feature
reconstruction.

| File | Features | Task label |
|------|----------|------------|
| site_A_vitals.csv | HR, SBP, DBP, Temp, SpO2, RespRate, GCS total (7 vars) | In-hospital mortality (binary, 48 h) |
| site_B_labs.csv | Glucose, pH, FiO2, CapRefill (4 vars) | Decompensation (binary, 24 h) |
| site_C_composite.csv | Height, Weight, MeanBP (3 vars) | Phenotyping (25 ICD codes, multi-label) |

---

## Step 5 — Align patient sets

```bash
python data_prep/psi_alignment.py \
    --site_a data/vertical_splits/site_A_vitals.csv \
    --site_b data/vertical_splits/site_B_labs.csv \
    --site_c data/vertical_splits/site_C_composite.csv \
    --output data/vertical_splits/aligned_patient_ids.csv
```

`psi_alignment.py` simulates a Private Set Intersection using SHA-256-hashed patient IDs,
computes the intersection across all three sites, and applies iterative stratification
(Sechidis et al., 2011) to ensure balanced train / val / test splits across all 27 binary
label signals (IHM + Decompensation + 25 phenotype codes).

---

## Step 6 — Verify

```bash
python data_prep/verify_workspace.py
```

Checks that all four task directories are non-empty, vertical splits are present,
aligned_patient_ids.csv covers at least 30,000 patients, feature columns do not overlap
between site files, and label balance is within acceptable range for each task.

---

## Notes on feature clipping

`vertical_split.py` applies hard clinical bounds to each feature before forward-fill /
backward-fill imputation. The bounds are tighter than the YerevaNN defaults in several
cases (e.g. SpO2 max = 100%, not 29,818%) because `clean_events()` normalises units but
does not enforce range caps during timeseries aggregation. Full bounds are documented in
`data_prep/vertical_split.py` and in the PRISM-VFL paper (Section 3.1).

## Synthetic mode (no MIMIC required)

All experiment scripts accept `--use_synthetic` to run a smoke test with randomly
generated data. This mode is useful for verifying installation but does not reproduce
any reported results.

```bash
python train.py --root . --rounds 5 --use-synthetic
```
